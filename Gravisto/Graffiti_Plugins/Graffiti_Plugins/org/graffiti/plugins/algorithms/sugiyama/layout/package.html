Description

This package contains algorithms, that implement the final layout-phase in the sugiyama-algorithm. They receive relative x- and y-coordinates from prior phases stored in the object NodeLayers in SugiyamaData and have to assign real x- and y-coordinates to the nodes.

Your implementation should not remove dummy-nodes from the graph, unless your algorithm provides special methods to remove these dummy-nodes and re-design their edges (e.g. use splines instead of bends).
If you'd add bends instead of dummy-nodes, you don't have to remove the dummy-nodes. The framework will take care of removing the dummy-nodes and adding bends to the original edge.

Your implementation should not save the coordinates by directly manipulating the CoordinateAttribute in the graphics-tree. You have to change or add the CoordinateAttribute coordinate in the sugiyama attribute-tree. The framework will take care of transferring these coordinates to the "real" coordinates inside the graphics-tree.

Requirements and information for implementation

In general, your implementation should extend AbstractAlgorithm, to save you some work and react to api-changes properly.

When you implement an algorithm for this phase, you have access to the object NodeLayers in SugiyamaData. This object stores each layer/level of the graph inside an ArrayList. The position of the nodes inside this array-list represents their order on the current level. You don't have to access attributes on the nodes to receive their order or level.

Your algorithm has to implement the following methods:

  • Getter/Setter for SugiyamaData
  • getName(): Return the name of your algorithm here - it is displayed in the algorithm-chooser
  • check(): Check if your algorithm can be applied to the current graph
  • execute(): Do your work here - compute the coordinates and write them into a CoordinateAttribute with the name "coordinate" inside the sugiyama attribute-tree.
  • setDefaultParameters(): Reset the parameters of your algorithm to their default state.

Optional

If it makes any sense for your algorithm, you can add support for animations to your algorithm. Please refer to the documentation of Animation-Objects for further information on how to support animations.
If you do support animations, you have to return true in the method supportsAnimation() inherited from AbstractAlgorithm.

Description

The sugiyama-plugin implements the interface EditorAlgorithm and has to supply its own ParameterDialog. This has to be done because of limitations in the default ParameterDialog of Gravisto.
Sugiyama is configured in two steps: In the first step (the first page of the dialog), the user has to select an algorithm for each phase. The dialog includes a search-button that lets the user search for additional algorithms that can be found somewhere in his classpath.
The utility-class FindPhaseAlgorithms is used to discover algorithms in the classpath. The available algorithms are stored into SugiyamaData and additionally into the filesystem through Java.Preferences, so the user does not have to search the entire classpath for algorithms each time he starts Gravisto.
Preferences-handling is done in the utility-class PreferencesUtil in the util-package.

The selected algorithms in the first page of the dialog cannot be passed to Sugiyama directly, but have to be stored into SugiyamaData. If the user clicks the next-button, all components in the dialog are removed and for each of the algorithms, a new ParameterEditPanel is added to the dialog. The dialog itself cannot be disposed and replaced with another one due to limitations in Gravisto.

The selected algorithms are stored into SugiyamaData, their configuration is stored through setParameters(). Sugiyama itself does not have to worry which algorithms had been selected or how they had been configured. All this information is stored in SugiyamaData or the correspondig algorithm.
